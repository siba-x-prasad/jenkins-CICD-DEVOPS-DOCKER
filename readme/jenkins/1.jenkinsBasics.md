This ensures a **faster**, **more reliable**, and **repeatable** development process.

Together, they form a **DevOps pipeline** â€” an automated process that helps teams deliver software **faster, safer, and more reliably**.

1. Code is written and committed.
2. Itâ€™s automatically built and tested.
3. If successful, itâ€™s deployed to staging or production.

## ğŸ” CI vs CD (Key Differences)

| Aspect         | Continuous Integration (CI)                  | Continuous Delivery (CD)                       | Continuous Deployment (CD)                           |
|----------------|----------------------------------------------|------------------------------------------------|------------------------------------------------------|
| **Goal**       | Merge code frequently and test automatically | Automate release preparation                   | Automate the entire release to production            |
| **Deployment** | Not included                                 | Manual trigger                                 | Fully automatic                                      |
| **Focus**      | Code integration & testing                   | Packaging & staging deployment                 | Full production deployment                           |
| **Risk**       | Detects early bugs                           | Reduces deployment risk                        | Delivers features faster but requires strong testing |
| **Example**    | Jenkins builds + tests code                  | Jenkins builds, tests, and prepares deployment | Jenkins builds, tests, and deploys live              |

# Example Jenkinsfile for CI/CD
```
pipeline {
agent any

    stages {
        stage('Build') {
            steps {
                echo 'Building the application...'
                sh 'mvn clean install'
            }
        }

        stage('Test') {
            steps {
                echo 'Running automated tests...'
                sh 'mvn test'
            }
        }

        stage('Staging Deployment') {
            steps {
                echo 'Deploying to staging environment...'
                sh './deploy-staging.sh'
            }
        }

        stage('Production Deployment') {
            when {
                branch 'main'
            }
            steps {
                echo 'Deploying to production automatically...'
                sh './deploy-production.sh'
            }
        }
    }
}
```
## Summary
- **Continuous Integration:** Automatically builds and tests code changes to ensure quality.
- **Continuous Delivery:**	Keeps the application in a deployable state and automates release preparation.
- **Continuous Deployment:** Fully automates deployment â€” code changes go live automatically after passing tests.
